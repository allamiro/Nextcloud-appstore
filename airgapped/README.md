# Nextcloud App Store — Air-Gapped Deployment

This directory contains everything needed to deploy the Nextcloud App Store in a disconnected (air-gapped) environment. It is generated by the package build workflow and can be deployed with either Docker Compose or Kubernetes.

## Directory Layout

```
airgapped/
├── docker-compose/
│   ├── docker-compose.airgapped.yml    # Core stack: postgres + appstore + nginx + fileserver
│   ├── docker-compose.nextcloud-test.yml # Optional: managed test Nextcloud overlay
│   └── .env.airgapped.example          # Environment template
├── k8s/
│   ├── 01-namespace.yaml
│   ├── 02-secrets.yaml
│   ├── 03-configmap.yaml
│   ├── 04-pvc.yaml
│   ├── 05-postgres.yaml
│   ├── 06-appstore.yaml
│   ├── 07-nginx.yaml
│   ├── 08-fileserver.yaml
│   ├── 09-nextcloud-test.yaml          # Optional: managed test Nextcloud
│   ├── 10-import-db-job.yaml           # One-shot DB import job
│   └── 11-configure-nextcloud-job.yaml # One-shot Nextcloud config job (same-cluster)
├── scripts/
│   ├── load-images.sh                  # Load saved Docker images into the runtime
│   ├── deploy-compose-airgap.sh        # Deploy with Docker Compose
│   ├── deploy-k8s-airgap.sh            # Deploy on Kubernetes
│   ├── configure-nextcloud-compose.sh  # Configure Docker Compose Nextcloud
│   ├── configure-nextcloud-k8s.sh      # Configure Kubernetes Nextcloud
│   ├── configure-nextcloud-ssh.sh      # Configure SSH/bare-metal Nextcloud
│   └── test-airgap.sh                  # Validate deployment
├── images/       # Docker images saved as .tar.gz (gitignored)
└── exports/      # DB dumps and app archives (gitignored)
```

## Quick Start

All commands below assume you are in the repository root and have copied `.env.example` to `.env`.

### 1. Load Images

```bash
./scripts/appstorectl.sh airgap load-images
```

### 2. Deploy (Docker Compose)

```bash
./scripts/appstorectl.sh airgap deploy compose
```

### 2. Deploy (Kubernetes)

```bash
./scripts/appstorectl.sh airgap deploy k8s
```

### 3. Configure Existing Nextcloud

```bash
# Docker Compose Nextcloud
./scripts/appstorectl.sh airgap configure-nextcloud external-compose

# Kubernetes Nextcloud
./scripts/appstorectl.sh airgap configure-nextcloud external-k8s

# SSH / bare-metal Nextcloud
./scripts/appstorectl.sh airgap configure-nextcloud external-ssh
```

### 4. Validate

```bash
./scripts/appstorectl.sh airgap test compose
# or
./scripts/appstorectl.sh airgap test k8s
```

## Optional: Managed Test Nextcloud

To spin up a temporary Nextcloud for testing (Docker Compose):

```bash
docker compose \
  -f airgapped/docker-compose/docker-compose.airgapped.yml \
  -f airgapped/docker-compose/docker-compose.nextcloud-test.yml \
  up -d
```

For Kubernetes:

```bash
kubectl apply -f airgapped/k8s/09-nextcloud-test.yaml
```

## TLS Trust

The App Store uses a self-signed CA chain generated by `k8s/generate-certs.sh`.

The root CA certificate is at `k8s/certs/root-ca.crt`.

**Docker Compose Nextcloud:**
```bash
docker cp k8s/certs/root-ca.crt <nextcloud-container>:/usr/local/share/ca-certificates/appstore-root-ca.crt
docker exec <nextcloud-container> update-ca-certificates
docker restart <nextcloud-container>
```

**Kubernetes Nextcloud:**
```bash
kubectl create configmap appstore-ca \
  --from-file=appstore-root-ca.crt=k8s/certs/root-ca.crt \
  -n <nextcloud-namespace>
# Then mount it and run update-ca-certificates in the Nextcloud pod.
```

**SSH Nextcloud:**
```bash
scp k8s/certs/root-ca.crt user@host:/tmp/appstore-root-ca.crt
ssh user@host "sudo cp /tmp/appstore-root-ca.crt /usr/local/share/ca-certificates/ && sudo update-ca-certificates"
```

**Nextcloud config.php (manual fallback):**
```php
'appstoreenabled' => true,
'appstoreurl' => 'https://appstore.local/api/v1',
```
